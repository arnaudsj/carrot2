DCS parameters (REST interface)

The parameters of the multipart HTTP POST request to the DCS service are explained below.

dcs.default.algorithm (optional)
Specifies the ID of the algorithm process to use for clustering. Default algorithm is used if not specified.
dcs.default.output (optional)
Specifies the desired output format. Available values are "json" and "xml" (default).
dcs.clusters.only (optional)
If equal to "true", the result will not echo back the input documents, only their references. It is recommended to set this parameter to "true".
any other key (optional)
Any parameters preceding the input stream are placed in the request context and are available for the components in the clustering chain. This is the way of setting configuration options for algorithms if they support them.
c2stream (required, last!)
An XML with documents to be clustered. The format of this XML is given in the example files. Note that this parameter must be the last one in the request as it causes the processing to begin immediately.
